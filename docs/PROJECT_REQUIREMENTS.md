# Project Requirements Document (PRD)

## CareerPilot AI — The Intelligent Career Copilot

---

## 1. Executive Summary

CareerPilot AI is a private, open-source, local-first AI SaaS platform that serves as a personal career assistant. Modern job seekers face an increasingly automated recruiting environment dominated by applicant tracking systems (ATS), complex skill landscapes, and intense interview processes. Proprietary resume optimization tools exist but are typically subscription-based, leak sensitive personal data, and rely on expensive external LLM APIs (e.g., OpenAI, Anthropic).

CareerPilot AI addresses these limitations by offering a robust enterprise-ready system built on a fully local and free AI stack: **FastAPI**, **PostgreSQL**, **Ollama (Qwen 2.5 3B Instruct)**, **spaCy**, and **Sentence Transformers**. On the frontend, a premium **Next.js** dashboard built with Tailwind CSS, shadcn/ui, and Framer Motion delivers a high-fidelity visual experience. The result is a secure, completely free-to-run career assistant that provides resume intelligence, semantic matching, interactive mock interviews, roadmap planning, and career analytics.

---

## 2. Problem Statement

1.  **High Costs:** Existing resume compilers and job matching sites charge monthly subscriptions ($20–$50/mo) for AI analysis.
2.  **Privacy Risks:** Uploading highly detailed resumes (containing addresses, emails, phone numbers, and full career histories) to cloud-based LLM APIs presents significant privacy vulnerabilities.
3.  **Keyword Matching Limitations:** Traditional ATS tools search for exact keyword matches, frequently disqualifying highly qualified candidates who describe their skills using different terminology (e.g., "React Native" vs. "Mobile App Development").
4.  **Ineffective Interview Prep:** Candidates lack low-friction, high-fidelity mock interview platforms that can test them on specific roles and assess their performance dynamically.

---

## 3. Objectives

*   **100% Free & Self-Hosted:** Provide a full-featured SaaS architecture that runs locally or on free cloud tiers (Vercel & Render) with zero paid API dependencies.
*   **Local AI Inference:** Execute advanced natural language processing (NLP), vector embeddings, and large language model (LLM) prompts entirely on consumer-grade local machines.
*   **Semantic Matching Accuracy:** Implement neural semantic searches to align job descriptions with resumes, achieving a cosine similarity match metric.
*   **High-Quality UX:** Deliver a glassmorphic dashboard matching modern SaaS aesthetics, avoiding placeholders and basic MVP designs.

---

## 4. Scope

### In-Scope
*   **Multi-Format Resume Upload:** Support parsing of standard PDF and DOCX files.
*   **Structured Data Extraction:** Extract skills, education, professional experience, projects, and contact metadata into standard JSON formats.
*   **ATS Screening & Analysis:** Identify missing keywords, formatting errors, section anomalies, and calculate an overall ATS compatibility score.
*   **Vector-Based Job Matching:** Match resumes to arbitrary job descriptions using embeddings generated by `Sentence Transformers` and calculated via Cosine Similarity.
*   **AI Resume Rewriter:** Suggest and apply bullet-point optimizations tailored to specific job targets.
*   **Interactive Mock Interview Console:** Text-based chat mock interviews tailored to specific roles, concluding with structured scoring and feedback.
*   **AI Career Roadmap:** Graphically display step-by-step career milestones to reach a target role.
*   **Persistent AI Chat Assistant:** System-wide chatbot maintaining context across different screens.
*   **Analytics Dashboard:** Graphic reporting of match history, skill gaps, and interview scores using Plotly.

### Out-of-Scope (Initial Phase)
*   **Automated Job Applying (Bots):** No auto-submission features to external job search portals.
*   **Recruiter Portal:** No multi-tenant admin portals for hiring managers or recruiters.
*   **Third-party Integrations:** No direct LinkedIn API syncs (due to platform auth restrictions).
*   **Payment Gateways:** No payment gateways or subscriptions (since the product is entirely free).

## 6. Functional Requirements

### FR-1: Authentication & User Profiles
*   Users must be able to sign up, log in, and log out securely.
*   JWT-based authentication tokens must be stored securely using HttpOnly cookies.
*   User profiles must persist parsed resumes, target roles, and past chat contexts.

### FR-2: Resume Intelligence & Parsing
*   The system must accept `.pdf` and `.docx` uploads up to 5MB.
*   Extract text and identify structural blocks: Contact, Work Experience, Education, Skills, Projects.
*   Use spaCy NLP models to tokenize, tag entities, and extract structured JSON schemas.

### FR-3: ATS Resume Analysis
*   Analyze resume formatting (missing contact information, inappropriate section headings, excessive layouts).
*   Generate an overall "ATS Score" (0–100%).
*   Provide actionable recommendations to resolve identified formatting issues.

### FR-4: Semantic Job Matching & Skill Gap Analysis
*   Provide an interface to paste target job descriptions.
*   Vectorise the resume skills and the job description skills.
*   Compute cosine similarity scores using local `Sentence Transformers`.
*   Generate a list of "Present Skills", "Missing Skills", and "Recommended Skills".

### FR-5: AI-Driven Resume Editor
*   Users can select specific bullet points in their resume experience.
*   The system prompts the local Ollama LLM to rewrite bullet points using the STAR method (Situation, Task, Action, Result).
*   Users can toggle between original and suggested text, applying the changes directly.

### FR-6: Mock Interview Console
*   Users can initiate a mock interview by selecting a target role and company.
*   The AI Assistant acts as the interviewer, generating 5 consecutive technical and behavioral questions.
*   The simulator records candidate text answers and provides a score card (1-10) with explanations and optimal answers upon completion.

### FR-7: AI Career Roadmap Generator
*   Given a user's current role and a target future role, the AI must chart a structured progression path.
*   The frontend must render this path as an interactive timeline, listing target skills, standard timelines, and target certification titles.

### FR-8: System-Wide AI Career Assistant
*   A persistent chat drawer accessible from any view in the portal.
*   Maintains conversation memory and answers career-related questions based on the user's uploaded resume data.

### FR-9: Analytics Dashboard
*   Plotly-based visualizations:
    *   Trend line of ATS scores over time.
    *   Radial radar charts comparing user skill levels vs. target job descriptions.
    *   Score tracking graphs of past mock interview scores.

## 7. Non-Functional Requirements

### Performance & Latency
*   **Resume Parsing:** Must complete text extraction and entity mapping in under 2 seconds.
*   **AI Inference (Local):** Local Ollama LLM response stream must begin under 500ms from request.
*   **Page Loading:** Next.js pages must load and reach interactive states in under 1 second locally.

### Security
*   **Data Encryption:** All credentials stored in PostgreSQL must be hashed using bcrypt.
*   **Stateless Security:** API requests must authorize using stateless JWT schemas.
*   **Data Privacy:** All resume processing, embeddings, and LLM text generation must occur entirely on local components (no external telemetry or vendor APIs).

### Usability & Design
*   **Modern Aesthetics:** Dark-themed glassmorphism dashboard styling matching premium modern SaaS metrics.
*   **Responsiveness:** Fluid styling that scales gracefully from mobile viewports to desktop resolutions.
*   **Micro-Animations:** Fluid transitions on page changes, drawer slides, and card hovers using Framer Motion.

### Portability & Free Hosting
*   **Deployment Compatibility:** The application must successfully deploy on free hosting tiers (Vercel for frontend static builds, Render for FastAPI backend services connected to a free PostgreSQL database).

---

## 8. Technical Constraints & Assumptions

### Technical Constraints
*   **Backend:** Python 3.12, FastAPI, SQLAlchemy, Alembic.
*   **Frontend:** Next.js (TypeScript), Tailwind CSS, shadcn/ui, Framer Motion.
*   **AI Engine:** Ollama (specifically running `qwen2.5:3b` parameters locally).
*   **Libraries:** spaCy (entity extraction), Sentence Transformers (embeddings), scikit-learn (cosine calculations).
*   **Database:** PostgreSQL (must use PostgreSQL-specific data types or standard relational models).
*   **No Paid APIs:** Under no circumstances should OpenAI, Anthropic, Pinecone, or other paid APIs be introduced.
*   **No Docker:** Dev and deployment setups must use native dependencies, environment files, and scripts without Docker wrappers.

### Assumptions
*   The user's local machine has at least 8GB-16GB of RAM to run Ollama's 3B model locally alongside the backend and frontend servers.
*   A running instance of Ollama is accessible at `http://localhost:11434` for local development.

---

## 9. Success Criteria

*   **Local Processing Validation:** Successful verification that no resume text, embeddings, or questions leave the client/server network loop to third-party endpoints.
*   **Successful Execution:** User can sign up, upload a resume, view structural charts, paste a job description, get a matching score, run a mock interview, and view an interactive roadmap without encountering database lockups or runtime exceptions.
*   **Premium Visual Impression:** Design scores high marks for professional layout, typography, dark mode balance, and interactive transition smoothness.
